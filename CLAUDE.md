# dotfiles

macOS dotfiles managed by chezmoi. Config files are copied (not symlinked) to `$HOME`.

# Chezmoi conventions
- `dot_` prefix becomes `.` in target (e.g., `dot_config/` -> `.config/`)
- `.tmpl` suffix means the file is a Go template processed by chezmoi
- `run_onchange_` scripts re-run when their content hash changes
- IMPORTANT: `chezmoi add` to add new files, or create manually with correct naming

# Template data
- `machineType` (`"personal"` | `"work"`): controls Brewfile casks in `dot_Brewfile.tmpl` and Claude plugins/marketplaces in `dot_claude/settings.json.tmpl`
# Claude Code config (`~/.claude/`)
Partially managed. Only user-created files are tracked:
- `CLAUDE.md` -- global instructions
- `settings.json` -- preferences, permissions, hooks (template: `enabledPlugins` and `extraKnownMarketplaces` vary by `machineType`)
- `skills/translate-ja/` -- translate-ja skill (Japanese to English)
- `skills/explain-en/` -- explain-en skill (English to Japanese with grammar notes)
- `skills/correct-en/` -- correct-en skill (English proofreading for learners)
- `statusline.py` -- fetched from [usedhonda/statusline](https://github.com/usedhonda/statusline) via `.chezmoiexternal.toml` (refreshed weekly)

NOT tracked: agents/, commands/, rules/, hooks/ (auto-installed by superpowers plugin), secrets, caches, session data.

After intentional changes to settings.json: edit `dot_claude/settings.json.tmpl` directly (it's a template, not a plain file)
To force-update statusline.py: `chezmoi apply --refresh-externals`

# Edit order rule
When changing any file managed by chezmoi:
1. Edit the chezmoi source file first (under `~/.local/share/chezmoi/`)
2. Run `chezmoi diff` and verify only the intended changes appear
3. Then run `chezmoi apply` to propagate changes to `$HOME`
Never edit `$HOME/.claude/` files directly when they are tracked here.

# What is NOT managed here
- Neovim `lazy-lock.json` -- generated by lazy.nvim, not tracked

# Git workflow
- Branch: feature branch -> PR -> `main`
- NEVER use `git -C` -- it bypasses hook-based permission controls
